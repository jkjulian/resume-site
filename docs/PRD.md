# 模块化简历网站 PRD（v1）

## 1. 项目目标
构建一个可维护的个人简历网站，通过“经历模块化 + 岗位定制”实现：
- 针对不同岗位快速生成定制简历页面
- 根据 JD（职位描述）给出关键词匹配与内容优化建议
- 支持中英文展示
- 支持导出 PDF

### 成功标准（v1）
- 30 分钟内可以从已有模块生成一版新岗位简历
- 至少支持 3 个岗位模板（backend / ai / data）
- 可访问独立链接，如 `/resume/backend`

---

## 2. 用户与使用场景
### 目标用户
- 仅自己使用（站点管理员也是自己）

### 核心场景
1. 录入/维护经历模块（项目、实习、技能、教育）
2. 选择目标岗位，自动筛选并排序模块
3. 粘贴 JD，获得关键词匹配与缺口提示
4. 生成岗位定制简历页并导出 PDF
5. 分享在线链接

---

## 3. 功能范围

## 3.1 模块库管理（必须）
- 新增/编辑/删除经历模块
- 字段：
  - `id`
  - `title`
  - `role`
  - `timeRange`
  - `tags`（如 backend / ai / data）
  - `skills`
  - `highlights`（结果导向，建议量化）
  - `metrics`（可选，数字化结果）
  - `language`（zh / en / bilingual）
  - `priority`（手动优先级）

## 3.2 岗位简历生成（必须）
- 输入：岗位类型（backend/ai/data）
- 处理：
  - 按标签筛选
  - 按规则排序（默认：影响力 > 时间新近 > 匹配度）
- 输出：岗位简历页面
  - 路由示例：`/resume/backend`

## 3.3 JD 优化助手（v1 可简化）
- 输入：JD 文本
- 输出：
  - 关键词列表（硬技能/软技能/业务词）
  - 已覆盖关键词（来自模块）
  - 缺失关键词
  - 建议补充的 2~3 条成果描述

## 3.4 多语言支持（必须）
- 语言切换：中文 / English
- 至少保证：页面 UI 文案 + 核心经历可双语显示

## 3.5 PDF 导出（必须）
- 在简历页提供“导出 PDF”
- 导出内容与当前岗位简历一致

---

## 4. 非功能需求
- 响应式：桌面优先，移动端可阅读
- 性能：首屏加载快（v1 不做重优化）
- 可维护：模块数据结构清晰，便于后续扩展
- 可部署：支持 Vercel 一键部署

---

## 5. 页面结构（v1）
1. `/` 首页
   - 个人简介
   - 岗位入口（backend / ai / data）
2. `/modules` 模块管理页
   - 模块列表 + 编辑入口
3. `/resume/[track]` 岗位简历页
   - 动态展示筛选后的模块
   - JD 输入区 + 匹配结果
   - 导出 PDF 按钮

---

## 6. 技术方案（v1）
- 框架：Next.js
- 样式：Tailwind CSS
- 数据：本地 JSON/TS（后续可接数据库）
- 部署：Vercel
- 原则：先可用，再优化
